> **文档性质：** 产品规划与模块路线图

## 1. 主程序定位

### 1.1 产品是什么

**文易传统文化** 是一个以传统文化符号（卦象、干支、奇门格局等）为载体的 **学习与自我整理工具**，帮助用户：

- 理解当前状态与节奏
- 获得可执行的行动提醒
- 进行情绪与决策前的自我反思

### 1.2 表达边界（全模块统一）

| 我们是什么 | 我们不是什么 |
|------------|--------------|
| 传统文化 **学习** | 精准 **预测** |
| **趣味解读** | **改命、化灾、转运** |
| **自我反思** 辅助 | 医疗、法律、投资、赌博建议 |
| **行动建议** 与节奏提醒 | **保证** 发财、复合、升职、考试结果 |

**统一免责声明基调：**

> 本内容仅供娱乐和传统文化参考，不构成任何预测、医疗、法律或投资建议。

所有模块（六爻、八字、奇门、今日运势）的文案、AI Prompt、分享卡片与海报 **必须** 遵守上述边界。

## 4. 每个模块的 MVP 边界

### 4.1 今日一卦（已实现，持续优化）

- 固定问题 + `category_id = 今日运势`
- 同 session 同本地日一条记录
- 复用六爻起卦与结果页
- 解读偏：今日节奏、行动提醒、情绪整理

### 4.2 六爻问事（已实现，主程序核心）

- 用户自选事项类型 + 自定义问题（敏感词拦截）
- 三枚硬币法，**不新增玄学算法**
- 本卦 / 变卦 / 动爻 / 免费 + 完整解读
- 起卦动画基于后端真实 `lines`

### 4.3 八字简析 MVP（P2，待开发）

**第一版只做：**

| 包含 | 不包含 |
|------|--------|
| 出生日期、时间（可选时辰 unknown） | 大运、流年、神煞大全 |
| 四柱（年柱、月柱、日柱、时柱） | 合婚、择日、起名 |
| 日主、五行强弱 **简析** | 精准事件预测 |
| 性格倾向、行动建议（模板 + AI） | 改运、化解、做法事 |

**输入：** 公历/农历择一（建议第一版仅公历 + 时辰选择）、性别（可选，用于文案中性化）。

**输出结构建议：** `summary` / `five_elements` / `day_master` / `tendency` / `action_steps` / `disclaimer`。

### 4.4 奇门高级问事 MVP（P3，待开发）

**第一版只做：**

| 包含 | 不包含 |
|------|--------|
| 用户问题 + 起局时间（默认当前） | 复杂九宫全盘交互 UI |
| 局势判断（偏保守表述） | 军事、赌博、投资具体建议 |
| 风险提醒、行动节奏 | 保证结果、定吉凶日期 |
| 免费摘要 + 完整解读（AI） | 与六爻混用同一张业务表 |

**定位：** 「高级问事」入口，明确标注学习用途，频次可限制（如每日 N 次）。

### 4.5 广告解锁（P1 mock → 后续真实）

| 阶段 | 行为 |
|------|------|
| **当前** | mock 按钮 / mock「看广告」弹窗，解锁完整解读 |
| **P1** | 抽象 `UnlockProvider`：`mock` \| `rewardedVideoMock` \| `rewardedVideo` |
| **流量主开通后** | 替换为微信激励视频 API，**不改** 解锁业务接口语义 |
| **不做** | 真实支付、订阅、次数包扣费（P4 再议） |

### 4.6 分享海报（已实现，各模块复用）

- 竖版 3:4，本地 Canvas
- 仅摘要，不含完整 AI 报告
- 小程序：搜索引导，**不伪造二维码**
- Web：结果页链接二维码（内测 IP / 后续 HTTPS 域名）

### 4.7 历史记录（已实现，扩展时按 module 筛选）

- 匿名 `session_key`
- 列表展示：模块名 + 日期 + 问题摘要
- 扩展后：`module_type` 字段区分六爻 / 八字 / 奇门 / 今日运势

## 5. 商业化结构

```text
用户进入模块
    ↓
免费解读（必得，合规短文案）
    ↓
完整解读（锁定）
    ↓
激励视频广告解锁（mock → 真实）
    ↓
完整解读展示 + 可选分享海报
```

| 层级 | 说明 | 当前 |
|------|------|------|
| **免费解读** | 短摘要，引导自我反思 | ✅ |
| **完整解读** | 结构化 JSON 报告 | ✅ |
| **激励视频解锁** | 观看广告后解锁 | mock ✅ → 真实 P1 |
| **分享传播** | 海报 / 分享卡片 | ✅ |
| **会员 / 次数包** | 订阅或按次付费 | ❌ 当前不做 |

**原则：**

- 不在未开通流量主前接真实广告 SDK
- 不在备案 / 支付资质未完成前接微信支付
- 商业化文案与解读正文 **分离**，避免「看广告改命」类表述

## 6. 技术架构建议

### 6.1 六爻：沿用现有 divination 流程

- 保持 `divination_record` + `divination_line` + `interpretation` + `unlock`
- 今日运势继续用 `daily_fortunes` 映射，**不** 与八字/奇门共用起卦表
- 起卦算法 **不重构**，新模块不复制三枚硬币逻辑到前端

### 6.2 新增八字 / 奇门前：通用 `analysis_records`

建议在 Phase D 设计统一报告表（名称示例）：

```text
analysis_records
  - id
  - session_id
  - module_type        -- liuyao | bazi | qimen | daily_fortune
  - category_id        -- 可选，关联 matter_category
  - input_payload      -- JSON：问题、出生信息、时间等
  - result_payload     -- JSON：四柱、局信息等结构化结果
  - unlock_status
  - status
  - created_at
```

**不要把八字、奇门硬塞进 `divination_record`：**

- 六爻有 `lines` / 本卦 / 变卦，模型清晰
- 八字是四柱五行，奇门是局盘信息，字段差异大
- 解读、解锁、历史可共用 **上层抽象**，底层 **分表或分 module_type**

### 6.3 解锁中心：`module_type` 感知

```text
POST /api/v1/analysis/{id}/unlock   -- 未来统一入口（示意）
或
POST /api/v1/divinations/{id}/unlock  -- 六爻继续可用，新模块并行新路由
```

解锁服务层建议：

- `UnlockService.Unlock(ctx, moduleType, recordID, sessionKey, unlockType)`
- `unlock_type`: `mock_ad` | `mock_button` | `rewarded_video` | `rewarded_video_mock`

### 6.4 AI Prompt 按模块拆分

| 模块 | Prompt 文件 / 包 | 要点 |
|------|------------------|------|
| 六爻 | 现有 `interpretation` + `ai/prompt.go` | 问事 / 今日运势分支 |
| 八字 | 新增 `bazi/prompt.go` | 禁止预测具体事件 |
| 奇门 | 新增 `qimen/prompt.go` | 局势 + 节奏，禁止保证结果 |

**Provider 逻辑（DeepSeek 调用）可共用**，仅 **user prompt 与 report schema** 按模块拆分。

### 6.5 前端 / 小程序 Tab 结构（规划）

```text
Tab: 今日 | 问事 | 扩展(八字/奇门) | 我的(历史/设置)
```

- P0 仅「今日 + 问事 + 历史」
- P2 起增加「八字」入口
- P3 起增加「奇门」入口（可折叠在「高级」下）

## 7. 审核与合规边界

### 7.1 全平台文案红线

**避免：**

- 「今天一定会发财 / 倒霉 / 分手 / 升职」
- 「保证复合」「保证转运」「化解灾祸」
- 医疗诊断、法律胜负、股票涨跌、彩票中奖
- 「天机已现」「命运注定」等恐吓或神化表达

**推荐：**

- 「更适合作为今日状态整理」
- 「你可以把它理解成一种自我反思的角度」
- 「行动建议仅供参考，不构成决策依据」

### 7.2 小程序审核注意

| 风险点 |  mitigation |
|--------|-------------|
| 玄学预测类目 | 强调学习、趣味、自我反思 |
| 诱导分享 | 分享文案克制，无利益承诺 |
| 广告 | mock 阶段不接真实广告；开通后合规配置流量主 |
| 用户隐私 | 出生信息最小化采集，说明用途 |
| 类目与简介 | 与落地页表述一致 |

### 7.3 模块差异化合规

| 模块 | 额外注意 |
|------|----------|
| 六爻 / 今日 | 已有敏感词拦截，保持 |
| 八字 | 不做「克夫」「必婚」等标签化断言 |
| 奇门 | 不做军事、赌博、投资具体指向 |
| 广告解锁 | 文案为「观看视频解锁完整解读」，非「看视频改运」 |

### 8.1 Phase A 验收要点（当前）

- [ ] 小程序：问事、今日、动画、解锁、海报、历史、分享
- [ ] Web/H5：同等核心路径（若并行维护）
- [ ] 后端：`ENABLE_DEBUG_ROUTES=false`（生产）
- [ ] 无密钥进 Git
- [ ] 文案合规抽检

### 8.2 Phase B–G 依赖关系

```text
Phase A (验收)
    → Phase B (广告 mock)
    → Phase C (八字方案)
    → Phase D (通用表) ──→ Phase E (八字开发)
    → Phase F1 (奇门后端 API) ──→ Phase F2 (奇门小程序)
    → Phase G (奇门 unlock / AI / 高级 UI，规划)
```

**建议：** Phase D 在 Phase E/G **之前** 完成，避免八字、奇门各建一套重复表结构。

## 10. 总结

| 维度 | 结论 |
|------|------|
| **主程序** | 传统文化学习 + 自我反思，非预测改命 |
